üë§ User Authentication APIs

POST /api/auth/user/register
Register a new user account. Creates a user with email and password authentication.
Request Body
Parameter Type Required Description
firstName String Yes User's first name
lastName String Yes User's last name
email String Yes User's email address (must be unique)
password String Yes Password (minimum 8 characters)
confirmPassword String Yes Password confirmation (must match password)
country String Optional User's country
role String Optional User role (default: "user")
subscription String Optional Subscription type: "free" or "premium" (default: "free")
Request Example
{ "firstName": "John", "lastName": "Doe", "email": "john.doe@example.com", "password": "password123", "confirmPassword": "password123", "country": "Nepal", "role": "user", "subscription": "free" }
Response
201 Created 400 Bad Request 500 Server Error
{ "success": true, "message": "User registered successfully", "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...", "user": { "id": "507f1f77bcf86cd799439011", "firstName": "John", "lastName": "Doe", "email": "john.doe@example.com", "country": "Nepal", "role": "user", "subscription": "free" } }
POST /api/auth/user/login
Login with email and password. Returns a JWT token for authenticated requests.
Request Body
Parameter Type Required Description
email String Yes User's email address
password String Yes User's password
Request Example
{ "email": "john.doe@example.com", "password": "password123" }
Response
200 OK 400 Bad Request 500 Server Error
{ "success": true, "message": "Login successful", "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...", "user": { "id": "507f1f77bcf86cd799439011", "firstName": "John", "lastName": "Doe", "email": "john.doe@example.com", "country": "Nepal", "role": "user", "subscription": "free" } }
POST /api/auth/user/google-login
Login or register using Google OAuth. Accepts Google ID token and creates/authenticates user.
Request Body
Parameter Type Required Description
idToken String Yes Google ID token from Google Sign-In
Request Example
{ "idToken": "eyJhbGciOiJSUzI1NiIsImtpZCI6IjEyMzQ1NiJ9..." }
Response
200 OK 400 Bad Request 401 Unauthorized 500 Server Error
{ "success": true, "message": "Google login successful", "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...", "user": { "id": "507f1f77bcf86cd799439011", "firstName": "John", "lastName": "Doe", "email": "john.doe@gmail.com", "country": null, "role": "user", "subscription": "free" } }

üß≠ Guide Authentication APIs

POST /api/auth/guide/register
Register a new guide account. Guides are trekking/tour guides with detailed profile information.
Request Body
Parameter Type Required Description
firstName String Yes Guide's first name
lastName String Yes Guide's last name
email String Yes Guide's email address (must be unique)
password String Yes Password (minimum 8 characters)
confirmPassword String Yes Password confirmation (must match password)
description String Yes Guide's description/bio
TBNumber String Yes Tourism Board registration number
trekAreas Array Yes Array of trek areas (non-empty)
experience Number Yes Years of experience (number)
education String Yes Educational background
languages Array Yes Array of languages spoken (non-empty)
ratePerDay Number Yes Rate per day (must be positive number)
certifications Array Yes Array of certifications (non-empty)
role String Optional Guide role (default: "guide")
Request Example
{ "firstName": "Sherpa", "lastName": "Tenzing", "email": "tenzing@example.com", "password": "password123", "confirmPassword": "password123", "description": "Experienced mountain guide with 10+ years", "TBNumber": "TB-12345", "trekAreas": ["Everest Base Camp", "Annapurna Circuit"], "experience": 10, "education": "Mountain Guide Certification", "languages": ["English", "Nepali", "Tibetan"], "ratePerDay": 5000, "certifications": ["IFMGA Certified", "Wilderness First Aid"] }
Response
201 Created 400 Bad Request 500 Server Error
{ "success": true, "message": "Guide registered successfully", "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...", "guide": { "id": "507f1f77bcf86cd799439012", "firstName": "Sherpa", "lastName": "Tenzing", "email": "tenzing@example.com", "role": "guide", "description": "Experienced mountain guide with 10+ years", "TBNumber": "TB-12345", "trekAreas": ["Everest Base Camp", "Annapurna Circuit"], "experience": 10, "education": "Mountain Guide Certification", "languages": ["English", "Nepali", "Tibetan"], "ratePerDay": 5000, "certifications": ["IFMGA Certified", "Wilderness First Aid"] } }

üîê Authentication

JWT Token: After successful login/registration, you'll receive a JWT token. Include this token in the Authorization header for protected routes:
Authorization: Bearer <your-jwt-token>
Token Expiration: Tokens expire after 7 days (configurable via JWT_EXPIRES_IN environment variable).
